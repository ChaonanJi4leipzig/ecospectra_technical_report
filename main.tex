\begin{abstract}
This technical report documents the current EcoSpectra traits-model experiments built around the reintegrated trait-centered data path. The report uses the project technical-report format, but replaces the original content with an implementation-oriented record of the current data contract, model stack, active validation gates, and experiment status. The current pipeline is organized into four layers: upstream data preparation, sample and contract construction, canonical model training, and ablations or extended comparisons. The active first-pass scope validates five traits---leaf area index, leaf mass per area, nitrogen, chlorophyll, and lignin---under a Copernicus HRL 2021 tree-cover-density mask. This 2026-05-09 update organizes accuracy-improvement work into five lanes: ERA5 climate context, ambient self-supervised temporal priors, spatial context, temporal-generalization diagnostics, and trait-relationship multitask learning. Completed results are reported only where manifests contain finished metrics; running or pending rows have blank result cells. The strongest completed positive signal remains ERA5-Land for RF and MLP, while the completed spatial-patch fast gate does not improve over the current sequence Transformer baseline.
\end{abstract}

\section{Introduction}

EcoSpectra develops trait-centered models for predicting vegetation functional traits from Earth-observation time series. The current work focuses on a reintegrated data path that rebuilds the sample and model contracts before promoting any new result to the full canonical model comparison. This is necessary because the trait-label source, forest mask, split policy, and model-ready sequence cache jointly define the scientific comparison: changing any of these components invalidates downstream model metrics.

The active pipeline is documented in the project memory under an A/B/C/D architecture. Layer A prepares upstream data sources, Layer B constructs sample manifests and model contracts, Layer C trains the canonical model stack, and Layer D runs ablations and extended comparisons. The current report captures the status as of 2026-05-09 and is intended as a reproducible technical snapshot rather than a final manuscript.

The report has three goals. First, it records the current data contract used by the reintegrated path. Second, it summarizes the experiments currently being run or interpreted. Third, it separates completed evidence from partial or pending work so that the next full 20-trait canonical run can be promoted only after the 5-trait gate is clean.

\section{Data and study area}

\subsection{Trait-label and predictor sources}

The trait-centered workflow uses sparse hyperspectral trait-label sources and dense temporal predictors. EnMAP remains the primary hyperspectral trait-label source; PRISMA v1 is implemented as an additional hyperspectral trait-label source under the project input tree. Sentinel-2 region zarrs provide the optical temporal predictor series \citep{drusch2012sentinel}. Optional climate context is derived from ERA5-Land nearest-point daily zarrs covering the 50 EnMAP-compatible sites and years 2022--2025 \citep{munozsabater2021era5land}.

The canonical forest mask is Copernicus HRL 2021 tree-cover density, with inclusion defined as $TCD \geq 30$. Sites 19, 30, and 47 are excluded by default because of land-cover coverage gaps. ForestPaths genus features remain ablation-only and are not part of the canonical mask.

\subsection{Current data contract}

The current B-layer contract is trait-centered. Each event is defined by site, year, spatial pixel, trait time, and source provenance where available. Sentinel-2 observations are assembled in an eight-slot temporal window using a zero-plus-mask missing-observation policy. A critical data rule is that valid pixels are not assumed to have supervised labels: the B2 manifest construction must retain a finite trait-label check because $valid\_mask=1$ can still coincide with all trait labels missing.

\subsection{Model families}

The canonical C-layer model stack consists of random forest, sequence MLP, and Transformer. Random forest provides the CPU tree baseline \citep{breiman2001random}; sequence MLP and Transformer consume the B8 temporal sequence cache, with Transformer following the self-attention sequence-model family introduced by \citet{vaswani2017attention}. LightGBM and XGBoost are active extended tree comparisons in D-layer experiments, not the current canonical C definition.

Fixed hyperparameters for future reintegrated-data experiments are taken from the cap2000 train20 hyperparameter searches recorded in the project memory. RF, LightGBM, and XGBoost use trait-specific defaults for the five gate traits. MLP and Transformer currently use the LMA-selected settings as neural defaults unless newer trait-specific searches supersede them.

\subsection{Evaluation}

Experiments use train, validation, and test partitions from the grouped split. Scalers, target transforms, imputers, and normalizers are fit on train data only. Metrics are reported in original trait scale through RMSE, normalized RMSE, and $R^2$. When a run is still active, this report records status from the run manifest instead of inferring missing metrics.

\section{Discussion}

The current evidence supports four practical conclusions. First, the reintegrated path is now sufficiently structured to be reported through a B-D-C gate: source matching, HRL filtering, grouped splits, and sequence-cache construction are treated as explicit contracts rather than implicit preprocessing details. Second, ERA5-Land context currently shows a useful paired signal for RF and MLP, but the Transformer result is weak and includes trait-level regressions, so ERA5 should be treated as a model-family-specific accuracy lever rather than a universal default. Third, the lightweight spatial-context gate does not support immediate expansion: the 3x3 patch CNN is worse than the sequence Transformer on both LAI and LMA. Fourth, the ambient-prior, temporal-diagnostic, and trait-relationship multitask lanes are still active and should not be interpreted until their final local summaries are pulled and checked.

The main limitation of this report is that it is a live technical snapshot. Some figures are generated from running summaries and should be regenerated after all manifest rows reach final status. In particular, the L2 ambient-prior and L5 trait-relationship results require completed JZ pulls, and the L4 temporal-generalization diagnostics require a consolidated notebook table before their results can be promoted from status tracking to scientific conclusions.

\section{Conclusion}

This EcoSpectra technical-report revision establishes a reproducible narrative for the current accuracy-improvement lanes. The immediate next revision should fill the blank L2, L4, and L5 result cells after final local summaries are available, regenerate prompt-standard figures, and then decide which lane merits promotion beyond cap2000 train20.
